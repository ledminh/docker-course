What is a Container?

A container is a lightweight, standalone, executable package that includes everything needed to run a piece of software: code, runtime, system tools, system libraries, and settings. Unlike virtual machines, containers share the host operating system's kernel, making them much more efficient in terms of resource usage and startup time.

üí° Key Concept

Think of containers as isolated processes running on your host machine. They appear to have their own file system, network, and process space, but they're actually sharing the underlying kernel with the host OS.

Container Architecture

Containers vs Virtual Machines

Understanding the difference between containers and virtual machines (VMs) is crucial for appreciating Docker's benefits. While both provide isolation, they achieve it in fundamentally different ways.

Virtual Machines Architecture

Key Differences

Aspect Containers Virtual Machines

Size Lightweight (MBs) Heavy (GBs)

How Containers Achieve Isolation

Containers use Linux kernel features to provide isolation without the overhead of a full virtual machine:

1. Namespaces

Namespaces provide isolated views of system resources. Each container gets its own:

  • PID namespace: Process IDs - containers see their own process tree
  • Network namespace: Network interfaces, IP addresses, routing tables
  • Mount namespace: File system mount points
  • IPC namespace: Inter-process communication resources
  • UTS namespace: Hostname and domain name
  • User namespace: User and group IDs

üîç Example

When you run ps aux inside a container, you only see the processes running in that container, not the processes from the host or other containers. This is thanks to PID namespaces!

2. Control Groups (cgroups)

Control groups limit and account for resource usage:

  • CPU: Limit CPU usage and set CPU priorities
  • Memory: Set memory limits and prevent containers from using too much RAM
  • Disk I/O: Control read/write speeds
  • Network: Bandwidth limitations

Hands-on Exercise: Verify Docker Installation

üìù Exercise 1: Check Docker Version and Info

Let's verify that Docker is installed and running on your system.

  1. Open your terminal or command prompt
  2. Run the following command to check Docker version:
Command
docker version

This command shows both the client and server (daemon) versions. You should see output like:

Expected Output

 Version:           24.0.7
 API version:       1.43
 Go version:        go1.20.10

Server: Docker Engine - Community

  Version:          24.0.7
  API version:      1.43 (minimum version 1.12)
  Go version:       go1.20.10
  Git commit:       311b9ff
  Built:            Thu Oct 26 09:07:41 2023
  OS/Arch:          linux/amd64
  1. Now run this command to get detailed information about your Docker installation:
Command
docker info

This displays comprehensive information including:

  • Number of containers (running, paused, stopped)
  • Number of images
  • Storage driver in use
  • Host kernel version
  • Operating system
  • Total memory and CPUs

Troubleshooting:

  • If you get "command not found", Docker might not be installed. Install Docker Desktop (Windows/Mac) or Docker Engine (Linux)
  • If you get "permission denied", you might need to use sudo on Linux or add your user to the docker group
  • On Windows/Mac, make sure Docker Desktop is running (check the system tray icon)

‚úÖ What You Learned

By running these commands, you've confirmed that both the Docker client (CLI) and Docker daemon (server) are running. The docker info command showed you details about the host system, including the kernel version that all containers on this host will share!

Summary

In this module, you learned the fundamental differences between containers and virtual machines. Key takeaways:

  • Containers are lightweight, isolated processes that share the host OS kernel
  • Virtual machines run complete operating systems on a hypervisor
  • Linux namespaces provide process-level isolation for containers
  • Control groups (cgroups) limit and manage resource usage
  • Containers are faster, more efficient, and more portable than VMs
  • Docker leverages these kernel features to provide containerization
